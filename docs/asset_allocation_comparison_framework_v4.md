# 资产配置策略定量对比报告 v4（方案一：指数收益序列对比）

---

# 一、框架目标

本报告用于系统性评估：

> 智能投顾 3.0 与 420_static 两套资产配置体系，在 35 类客户画像下，基于统一指数收益序列的策略层配置能力差异。

评估维度覆盖：**收益能力、风险水平、风险收益比（夏普）、风险匹配度（Δσ）**。

本框架不预设智能投顾 3.0 等于某一种具体优化算法。智能投顾 3.0 可以是 MVO、风险平价、带规则约束的优化、目标风险组合、专家规则 + 优化混合框架——本评估比较的是"策略体系结果"，而不是单一算法标签。

---

# 二、资产体系（统一口径）

四大类资产：

| 资产类 | 定义 | 指数口径（Total Return） |
|--------|------|--------------------------|
| CASH | 现金类 | CBA02201 |
| BOND | 固收类 | CBOND_NEW_COMPOSITE_WEALTH |
| EQUITY | 股票类 | CSI300_TR（沪深300全收益） |
| ALT | 另类 | 70% × AU9999 + 30% × NHCI |

月收益率计算：`R_i(t) = P_t / P_{t-1} - 1`

要求：智能投顾 3.0 与 420_static 必须使用一致的四大类资产口径，不允许一套方案使用不同资产定义进行比较。

# 三、风险锚体系（统一裁判层）

## 3.1 核心定位

风险锚体系不是用来规定某一类客户"标准上应该买什么组合"，而是用来为不同客户画像定义**统一风险容器（Risk Container）**，并作为智能投顾 3.0 与 420_static 的共同裁判层。

概括：**先定义客户能承受多少风险，再评估策略是否把风险用对了。**

在智能投顾 3.0 的多引擎架构下，风险锚体系承担三项职责：

- **事前约束（ex-ante constraint）**：为不同策略引擎提供统一风险边界
- **事后评估（ex-post evaluation）**：对不同策略结果进行统一适配性评估
- **横向可比（cross-engine comparability）**：让目标函数不同的策略能够在同一把尺子下比较

## 3.2 核心变量

对于每一个客户画像（risk_level × life_stage），定义：

```text
σ_cap   ：风险上限 = σ_stage_max × m_risk
σ_mid   ：风险中枢 = 0.8 × σ_cap（默认占用 80% 风险预算，预留缓冲）
σ_min   ：风险下限 = 0.6 × σ_cap（低于此值说明风险预算使用不足）
MaxDD   ：最大回撤容忍度（风险红线）
```

其中：

- **σ_stage_max**：由生命周期决定的基础风险上限
- **m_risk**：由风险等级决定的风险放大/收缩系数
- **σ_mid**：客户画像的目标风险中枢，用于评估组合是否过激进或过保守

## 3.3 生命周期波动率上限（σ_stage_max）

### 3.3.1 总结原则

本表**不是行业统一标准值表**，而是参考 Vanguard、BlackRock 等海外主流 Target-Date Fund 的 **glide path** 方法论，结合中国市场波动特征、适当性约束与内部回测结果形成的本地化风险预算。

核心原则：**生命周期越往后，组合风险预算通常越下降。**

这背后主要基于两条相对客观的逻辑：

1. **年轻阶段 human capital 更高**：未来劳动收入现金流更长，金融资产可以承受更高波动
2. **生命周期后段投资 horizon 更短**：离退休或资金兑现时点更近，承受大回撤后的修复时间更少

因此，刚毕业阶段的风险上限高于子女成年或退休阶段，并不是主观偏好判断，而是将行业通行的生命周期投资逻辑参数化到本项目的风险锚体系中。

【术语注释】
- **Target-Date Fund（目标日期基金）**：围绕未来目标年份进行长期资产配置的基金。
- **glide path（风险路径）**：组合风险随人生阶段或目标日期变化而逐步下调的配置路径。

### 3.3.2 设定说明

- **Vanguard / BlackRock 的启发**：行业共识是“年轻时偏增长、临近退休时偏稳健”，但并不存在某一人生阶段必须对应某个统一波动率数值
- **本项目的处理方式**：不直接照搬海外公开产品参数，而是吸收其 glide path 方法论，再转译为本项目可执行、可回测、可比较的年化波动率上限
- **因此需要强调**：0.16、0.15、0.14、0.12、0.08 等数值属于内部参数化结果，不应表述为行业固定标准

### 3.3.3 生命周期参数表

| 生命周期 | 代码 | σ_stage_max | 说明 |
|---------|------|-------------|------|
| 刚毕业 | S1 | 0.16 | 积累早期，human capital 高，风险预算较高 |
| 单身青年 | S2 | 0.15 | 仍以长期积累为主 |
| 二人世界 | S3 | 0.14 | 家庭责任上升，风险适度收敛 |
| 小孩学前 | S4 | 0.14 | 现金流支出上升，保持中等风险预算 |
| 小孩成年前 | S5 | 0.13 | 教育与家庭支出压力提升 |
| 子女成年 | S6 | 0.12 | 更接近财富稳态与退休准备阶段 |
| 退休 | S7 | 0.08 | 以保值、回撤控制与流动性管理为主 |

## 3.4 风险等级乘数（m_risk）与回撤容忍度

生命周期决定的是风险预算的**基础上限**，风险等级决定的是客户在该生命周期下愿意承受风险的**强弱程度**。

因此，本框架采用“生命周期定上限、风险等级定强度”的两层结构：

| 风险等级 | m_risk | MaxDD | 说明 |
|---------|--------|-------|------|
| C1 | 0.45 | -5% | 保守型 |
| C2 | 0.60 | -10% | 稳健型 |
| C3 | 0.75 | -15% | 平衡型 |
| C4 | 0.90 | -20% | 进取型 |
| C5 | 1.00 | -25% | 激进型 |

其中：

- **m_risk** 用于在同一生命周期内部拉开不同风险等级的风险预算
- **MaxDD** 用于刻画极端情形下的回撤红线，与年化波动率共同构成“常态风险 + 极端风险”双层约束

## 3.5 计算示例

**C3 / 二人世界（S3）**：σ_cap = 0.14 × 0.75 = 0.105 → σ_mid = 0.084 → σ_min = 0.063

**C1 / 退休（S7）**：σ_cap = 0.08 × 0.45 = 0.036 → σ_mid = 0.0288 → σ_min = 0.0216

**C5 / 刚毕业（S1）**：σ_cap = 0.16 × 1.00 = 0.16 → σ_mid = 0.128 → σ_min = 0.096

## 3.6 核心设计原则

1. **先定风险容器，再评估配置结果**——风险锚体系定义的是客户适配边界，不是标准答案组合
2. **生命周期与风险等级分层解耦**——生命周期决定基础上限，风险等级决定承受强度
3. **风险匹配优于风险最低**——波动率最低不等于最优，最接近 σ_mid 的组合才更适配客户
4. **行业给方法论，本项目给参数化落地**——方向参考 Target-Date Fund，数值来自本地化校准与内部治理
5. **σ 与 MaxDD 分工明确**——σ刻画常态风险预算，MaxDD刻画极端风险红线，避免语义混淆

# 四、对比方案设计

## 4.1 定义

本方案用于回答：

> 在不受具体产品噪音、费用、成立时点等因素影响时，智能投顾 3.0 与 420_static 两套策略体系本身的配置能力谁更优。

## 4.2 对比对象

- **策略A**：智能投顾 3.0（AI 大模型生成配置权重）
- **策略B**：420_static（原 420 体系静态权重）

两者统一映射到四大类资产指数收益序列上进行历史回测。

## 4.3 客户覆盖

35 类客户画像 = 5 个风险等级（C1–C5）× 7 个生命周期阶段（S1–S7），每类画像对应唯一 `profile_id`（如 `C3_S3`）。

适当性约束（eligibility）：

- C1：仅 CASH / BOND
- C2：CASH / BOND / ALT（无 EQUITY）
- C3–C5：CASH / BOND / EQUITY / ALT（全部四类）

## 4.4 时间窗口

1 年 / 3 年 / 5 年 / 10 年 / 20 年

均取历史月频数据末尾 N 个月进行回测。

## 4.5 回测逻辑

**Step 1**：对 35 类客户分别获取两套策略的四大类资产配置权重（静态，买入持有）

**Step 2**：统一使用指数收益序列计算组合月收益

```text
R_p(t) = Σ (w_i × R_i(t))
```

**Step 3**：按不同时间窗口计算绩效指标

# 五、核心指标定义

## 5.1 绩效指标

| 指标 | 公式 | 说明 |
|------|------|------|
| 年化收益 | `(∏(1+R_t))^(12/T) - 1` | 月度复利年化 |
| 年化波动 | `std(R_t, ddof=1) × √12` | 月度标准差年化 |
| 夏普比率 | `(Return_ann - 0.02) / Vol_ann` | 无风险利率 = 2% |
| 最大回撤 | `max(peak - trough) / peak` | 累计收益序列中最大峰谷跌幅 |

## 5.2 风险匹配指标（基于风险锚体系）

| 指标 | 公式 | 说明 |
|------|------|------|
| Δσ | `Vol_ann - σ_mid` | 组合波动与客户风险中枢偏差；正值偏激进，负值偏保守 |
| \|Δσ\| | `\|Vol_ann - σ_mid\|` | 风险匹配度核心指标——越小越匹配客户 |

关键理念：**不是波动越低越好，而是越接近 σ_mid 越好。** σ_mid 是每个客户画像基于风险锚体系计算得出的目标波动率，代表该客户应有的风险预算占用水平。

## 5.3 对比指标

| 指标 | 公式 | 说明 |
|------|------|------|
| 收益差 | `Return_3.0 - Return_420` | 正值 = 3.0 收益更高 |
| 波动差 | `Vol_3.0 - Vol_420` | 正值 = 3.0 波动更高 |
| 夏普差 | `Sharpe_3.0 - Sharpe_420` | 正值 = 3.0 风险收益比更优 |
| 收益胜率 | `P(Return_3.0 > Return_420)` | 35 个画像中 3.0 收益更高的占比 |
| 夏普胜率 | `P(Sharpe_3.0 > Sharpe_420)` | 35 个画像中 3.0 夏普更高的占比 |
| 风险匹配胜率 | `P(\|Δσ_3.0\| < \|Δσ_420\|)` | 35 个画像中 3.0 风险匹配更优的占比 |

## 6.2 结果解读

### 收益维度

3.0 在所有时间窗口上的平均年化收益均高于 420_static，收益差从短期（1y: +4.72pp）到长期（20y: +0.65pp）呈收敛趋势。收益胜率在 1y–10y 窗口稳定在 85.7%–88.6%，即绝大多数客户画像下 3.0 的收益均优于 420_static。20y 窗口收益胜率回落至 77.1%，但仍保持多数优势。

### 风险收益比（夏普）维度

1y–10y 窗口中，3.0 的平均夏普均高于 420_static，说明在承担更多波动的同时也获取了更多超额收益。1y 窗口夏普胜率最高（88.6%）。20y 窗口 3.0 平均夏普（0.78）低于 420_static（0.86），夏普胜率降至 45.7%——这是 3.0 唯一在夏普维度未胜出的窗口，反映出超长期回测中 3.0 较高的波动暴露对夏普的拖累。

### 风险匹配度（|Δσ|）维度

**在所有五个时间窗口中，3.0 的平均 |Δσ| 均小于 420_static**，即 3.0 的组合波动平均更接近每个客户的风险中枢 σ_mid。

- 1y 窗口：3.0 平均 |Δσ| = 2.84%，420_static = 3.93%，差距最为显著（-1.09pp）
- 3y–10y 窗口：差距在 0.49pp–0.53pp 之间，3.0 持续更优
- 20y 窗口：差距收窄至 0.12pp，但 3.0 仍更优

风险匹配胜率在 1y 窗口达 71.4%（25/35 个画像 3.0 更匹配），20y 窗口为 65.7%（23/35），3y–10y 窗口为 57.1%（20/35）。

**关键结论**：3.0 不仅在收益上优于 420_static，在"是否适合客户"这一维度上同样更优——它的配置结果更接近每个客户画像基于风险锚体系计算得出的风险中枢目标。

### 20 年窗口的特殊性

20y 窗口是唯一一个 3.0 夏普低于 420_static 的时段。两者波动均显著放大（3.0: 8.74%, 420: 7.89%），因为 20y 数据覆盖了 2008 年金融危机等极端市场周期。在此窗口中，3.0 的更高权益/另类暴露导致波动放大幅度大于收益增幅，从而拖累夏普。但即便如此，3.0 的风险匹配胜率（65.7%）仍为所有窗口中最高之一，说明从客户适配性角度看 3.0 在极端周期中的表现仍然合理。

# 七、风险匹配深度分析

## 7.1 |Δσ| 的含义

|Δσ| 是本评估体系中的核心适配性指标。它度量的不是"风险高低"，而是"风险是否匹配客户"：

```text
|Δσ| = |Vol_ann - σ_mid|
```

- |Δσ| 小 → 组合波动接近客户风险中枢 → 风险预算使用合理
- |Δσ| 大且 Δσ > 0 → 组合波动超过风险中枢 → 偏激进，风险预算超支
- |Δσ| 大且 Δσ < 0 → 组合波动远低于风险中枢 → 偏保守，风险预算闲置

## 7.2 两套策略的风险匹配模式

从 35 个画像的 detail 数据可以观察到以下模式：

**C1（保守型）客户**：两套策略的组合波动均远低于 σ_mid（σ_mid 在 2.88%–5.76% 之间，而实际波动多在 0.5%–1.5%），|Δσ| 均较大。这说明在极保守客户上，两套策略都存在风险预算严重闲置的情况，受限于适当性约束（C1 仅可配 CASH/BOND）。

**C2（稳健型）客户**：3.0 的 |Δσ| 显著优于 420_static，尤其是 S1–S3 阶段。例如 C2_S2 在 1y 窗口中 3.0 的 |Δσ| 仅为 0.39%（几乎完美匹配），而 420_static 为 5.05%。

**C3–C5（平衡/进取/激进型）客户**：3.0 在中短期窗口（1y–5y）普遍展现更优的风险匹配，但在 20y 窗口中部分高风险画像（如 C4_S1、C5_S1）的波动显著放大，|Δσ| 有所恶化。

## 7.3 广度 vs 深度

- **广度（win_rate_abs_delta_sigma）**：看多少个画像 3.0 风险匹配更优——1y: 71.4%, 20y: 65.7%
- **深度（mean_abs_delta_sigma）**：看平均偏离程度——3.0 在所有窗口上均更低

两个维度共同说明 3.0 在风险匹配上的优势既有广度又有深度。

# 八、统一口径要求

## 8.1 客户口径

同样的 35 类客户画像（C1–C5 × S1–S7），同样的风险等级与生命阶段定义。

## 8.2 资产口径

同样的四大类资产定义，同样的指数收益序列（Total Return 口径）。

## 8.3 风险口径

统一使用风险锚体系的 σ_mid 作为每个客户画像的风险中枢，统一计算 |Δσ| = |Vol_ann − σ_mid|。

## 8.4 回测规则

- 频率：月频
- 再平衡：买入持有（静态权重，不再平衡）
- 交易成本：忽略（方案一为策略层指数对比）
- 无风险利率：2%（固定）
- 缺失值处理：无长期前向填充

---

# 九、体系边界与局限

1. 风险锚体系以年化波动率为核心风险度量，对尾部风险、流动性风险、信用风险的刻画有限
2. σ_stage_max 基于历史统计经验与行业共识设定，在市场范式切换时需重新评估
3. |Δσ| 为事后指标，评估效果依赖观察窗口长度；窗口过短时稳定性不足
4. 本方案为策略层（指数）对比，不反映真实产品落地时的费用、滑点、产品替换等因素
5. 420_static 权重目前仍为 mock 数据，待替换为真实数据后结论需重新验证

# 十、总结

本报告基于统一的四大类资产指数收益序列，对智能投顾 3.0 与 420_static 在 35 类客户画像、5 个时间窗口下进行了系统性策略层对比，并引入风险锚体系作为客户适配性的统一裁判层。

核心发现：

1. **收益优势明显**：3.0 在 1y–20y 所有窗口上平均年化收益均高于 420_static，收益胜率 77%–89%
2. **夏普整体更优**：1y–10y 窗口 3.0 夏普均更高，仅 20y 窗口因长期波动放大略有落后
3. **风险匹配更优**：3.0 的平均 |Δσ| 在所有窗口上均小于 420_static，说明 3.0 的配置结果更接近每个客户的风险中枢目标——这是从"收益比较"向"适配性比较"升级后最关键的结论

> 在统一指数口径和风险锚裁判体系下，智能投顾 3.0 在绝大多数客户画像和时间窗口上，不仅获取了更高的收益，而且更匹配客户的风险预算目标——配置结构整体优于 420_static。
