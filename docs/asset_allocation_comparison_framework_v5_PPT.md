# 第一页、SAA 的重要性：资产配置决定了长期结果的底盘

## 1.1 什么是 SAA

SAA（Strategic Asset Allocation，战略资产配置）是指：先确定组合在现金、固收、股票、另类等**大类资产**上的长期配置框架，再在该框架内做产品选择与动态调整。

它回答的是一个更底层的问题：**客户的钱，应该先按什么风险收益结构分配到不同资产上。**

## 1.2 为什么 SAA 是核心

在财富管理场景中，长期业绩主要不是由单只产品挑选决定，而是由以下四个底层因素决定：

1. **大类资产配置**：组合的收益来源首先由股、债、现金、另类的比例决定。
2. **资产间相关性**：不同资产之间是否能分散风险，决定了组合波动和回撤表现。
3. **风险匹配度**：组合风险是否与客户画像匹配，决定了方案是否适当、可持有、可执行。
4. **投资期限**：不同生命周期客户的风险承受能力和资金使用时点不同，决定了风险预算应当不同。

因此，比较两套资产配置策略，不能只看“谁收益更高”，而要看：

- 是否在同一大类资产口径下比较；
- 是否在同一风险锚体系下比较；
- 是否对同一批客户画像比较；
- 是否同时比较收益、风险、风险匹配与极端回撤。

## 1.3 核心比较问题

> 在统一四大类资产口径、统一客户画像、统一风险锚体系后，3.0 策略是否能在**不显著偏离客户风险预算**的前提下，取得更优的长期风险收益表现。

# 第二页、统一比较口径：先对齐四大类资产，再对齐风险锚体系

## 2.1 对齐四大类资产

为了保证两套策略具备可比性，本报告先将所有底层产品与策略映射到统一的四大类资产框架。

| 资产类 | 定义 | 指数口径（Total Return） |
|--------|------|--------------------------|
| CASH | 现金类 | 中债货币市场基金可投资债券财富(总值)指数 |
| BOND | 固收类 | 中债新综合指数-财富指数 |
| EQUITY | 股票类 | 沪深300全收益指数 |
| ALT | 另类 | 70% × AU9999（上海黄金交易所 Au99.99） + 30% × NHCI（南华商品指数） |

## 2.2 对齐风险锚体系

在统一四大类资产后，还必须统一“什么叫适合这个客户”的风险标准，即风险锚体系。

本框架采用“两层风险锚”结构：

- **生命周期**决定风险预算的基础上限；
- **风险等级**决定客户在该生命周期下愿意承担风险的强弱程度。

换句话说：

> 生命周期定上限，风险等级定强度。

# 第三页、风险锚体系：行业主流做法的本地化落地

## 3.1 生命周期参数表

| 生命周期 | 代码 | σ_stage_max | 说明 |
|---------|------|-------------|------|
| 刚毕业 | S1 | 0.16 | 积累早期，human capital 高，风险预算较高 |
| 单身青年 | S2 | 0.15 | 仍以长期积累为主 |
| 二人世界 | S3 | 0.14 | 家庭责任上升，风险适度收敛 |
| 小孩学前 | S4 | 0.14 | 现金流支出上升，保持中等风险预算 |
| 小孩成年前 | S5 | 0.13 | 教育与家庭支出压力提升 |
| 子女成年 | S6 | 0.12 | 更接近财富稳态与退休准备阶段 |
| 退休 | S7 | 0.08 | 以保值、回撤控制与流动性管理为主 |

## 3.2 风险等级乘数（m_risk）与最大回撤容忍度

| 风险等级 | m_risk | MaxDD | 说明 |
|---------|--------|-------|------|
| C1 | 0.45 | -5% | 保守型 |
| C2 | 0.60 | -10% | 稳健型 |
| C3 | 0.75 | -15% | 平衡型 |
| C4 | 0.90 | -20% | 进取型 |
| C5 | 1.00 | -25% | 激进型 |

其中：

- **m_risk** 用于在同一生命周期内部拉开不同风险等级的风险预算；
- **MaxDD** 用于刻画极端情形下的回撤红线；
- 年化波动率与最大回撤共同构成“常态风险 + 极端风险”的双层约束。

## 3.3 核心计算变量

对于每一个客户画像（risk_level × life_stage），定义：

```text
σ_cap   ：风险上限 = σ_stage_max × m_risk
σ_mid   ：风险中枢 = 0.8 × σ_cap
σ_min   ：风险下限 = 0.6 × σ_cap
MaxDD   ：最大回撤容忍度（风险红线）
```

其中：

- **σ_cap**：客户画像允许的风险上限；
- **σ_mid**：客户画像的目标风险中枢；
- **σ_min**：风险预算使用不足的下界；
- **MaxDD**：极端回撤情形下的容忍红线。

## 3.4 计算示例

**C3 / 二人世界（S3）**：σ_cap = 0.14 × 0.75 = 0.105 → σ_mid = 0.084 → σ_min = 0.063

**C1 / 退休（S7）**：σ_cap = 0.08 × 0.45 = 0.036 → σ_mid = 0.0288 → σ_min = 0.0216

**C5 / 刚毕业（S1）**：σ_cap = 0.16 × 1.00 = 0.16 → σ_mid = 0.128 → σ_min = 0.096

## 3.5 为什么说这套风险锚体系是行业主流设计

这套参数**不是行业统一标准值表**，但其方法论是行业主流。

它参考的是 Vanguard、BlackRock 等海外主流 Target-Date Fund 的 **glide path** 设计思路，并结合中国市场波动特征、适当性管理要求与本项目回测结果进行本地化落地。

其核心逻辑并不主观，而是遵循两条通行原则：

1. **年轻阶段 human capital 更高**：未来劳动收入现金流更长，金融资产可承受更高波动；
2. **生命周期后段投资 horizon 更短**：越接近退休或资金兑现时点，越难承受大回撤后的修复周期。

因此，本报告中的风险锚体系，本质上是把行业通行的生命周期投资逻辑，转成适用于本项目的、可量化、可回测、可解释的参数体系。

【术语注释】
- **Target-Date Fund（目标日期基金）**：围绕未来目标年份进行长期资产配置的基金。
- **glide path（风险路径）**：组合风险随人生阶段或目标日期变化而逐步下调的配置路径。
