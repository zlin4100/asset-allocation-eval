# 一、风险锚体系定义（核心思想）

本系统不预设“标准资产配置组合”，而是为每类客户定义：

> **风险承受区间（Risk Container）**

并通过组合实际波动与风险中枢的偏差，评估策略是否匹配客户风险。

在智能投顾 3.0 的多引擎架构下（如 MVO / 风险平价 / 目标风险 / 规则+优化混合 / 专家覆盖），本体系的定位不是服务某一个优化器，而是作为**策略无关的统一裁判层**：

- **事前约束（ex-ante constraint）**：为不同引擎提供统一风险边界
- **事后评估（ex-post evaluation）**：对不同引擎产出的组合进行统一适配性评估
- **跨引擎可比性**：让目标函数不同的策略也能在同一把尺子下横向比较

---

# 二、核心变量定义

对于每一个客户画像（risk_level × life_stage），定义：

```text
σ_cap   ：风险上限（最大可承受波动）
σ_mid   ：风险中枢（风险预算目标占用率对应的目标波动）
σ_min   ：风险预算闲置阈值（低于该值说明风险预算使用不足）
MaxDD   ：最大回撤容忍度（风险红线）
```

---

# 三、计算公式（核心）

## 1️⃣ 生命周期风险上限

```text
σ_stage_max = f(life_stage)
```

---

## 2️⃣ 风险等级调节

```text
σ_cap = σ_stage_max × m_risk
```

---

## 3️⃣ 风险中枢

```text
σ_mid = 0.8 × σ_cap
```

说明：

- 0.8 为默认经验系数（可调）
- 不表示某个单一优化器的“理论最优点”
- 表示：组合目标波动默认占用 80% 的风险预算，为市场环境恶化预留缓冲

---

## 4️⃣ 风险区间

```text
σ_min = 0.6 × σ_cap
σ_max = σ_cap
```

说明：

- `σ_min` 不是“风险越低越差”的含义
- 而是用于识别组合是否长期明显低于客户应承担的风险水平
- 若组合波动长期低于 `σ_min`，可视为风险预算闲置，可能牺牲长期收益

---

## 5️⃣ 风险匹配偏差（评估核心指标）

```text
Δσ = Vol_ann - σ_mid
```

```text
|Δσ| 越小 → 风险匹配越好
```

👉 该指标为整个评估体系核心。

---

# 四、生命周期波动率上限（σ_stage_max）

说明：

- `σ_stage_max` 不是凭经验随意设定，而是用于刻画不同生命周期阶段的长期风险承载上限
- 其设定可参考海外主流 Target-Date / Lifecycle Fund 的 **glide path** 思路，即：随着客户逐步接近退休，组合风险暴露应整体单调下降
- 其中，Vanguard、BlackRock 等海外主流 Target-Date Fund 管理机构的 glide path，可作为“生命周期 → 风险水平”映射关系的行业参考锚点
- 本文档中的 `σ_stage_max` 并非直接照搬海外养老金产品权重路径，而是参考其生命周期风险递减逻辑，并结合中国市场资产波动特征与零售财富客户特征进行本地化校准

| 人生阶段 | σ_stage_max | 说明 |
|---------|-------------|------|
| 刚毕业 | 0.16 | 收入增长期，可承受高波动 |
| 单身青年 | 0.15 | |
| 二人世界 | 0.14 | |
| 小孩学前 | 0.14 | 支出上升，风险略收敛 |
| 小孩成年前 | 0.13 | 教育支出压力 |
| 子女成年 | 0.12 | 财富稳态 |
| 退休 | 0.08 | 以保值为主 |

### 行业参考（Vanguard / BlackRock glide path）

在海外养老金与目标日期基金（Target-Date Fund）实践中，主流机构通常会采用一条随生命周期推进而逐步降低风险资产暴露的 **glide path**：

- 年轻阶段：权益占比较高，允许更高波动
- 临近退休：逐步提升债券及低波动资产占比
- 退休后：整体风险预算继续收敛，更强调回撤控制与购买力保护

因此，从方法论上看，glide path 提供的不是某一个具体权重答案，而是一种被广泛验证的行业共识：

> 生命周期越靠后，客户可承受的组合风险上限应越低。

本文将这一行业共识进一步抽象为 `life_stage → σ_stage_max` 的风险锚表达，用于适配智能投顾 3.0 的多引擎架构。

# 五、风险等级乘数（m_risk）

| 风险等级 | m_risk | 说明 |
|---------|--------|------|
| C1 | 0.45 | 保守型，低波动但仍需收益 |
| C2 | 0.60 | 稳健型 |
| C3 | 0.75 | 平衡型 |
| C4 | 0.90 | 进取型 |
| C5 | 1.00 | 激进型 |

---

# 六、最大回撤容忍度（MaxDD）

仅由风险等级决定（不随生命周期变化）：

| 风险等级 | MaxDD |
|---------|-------|
| C1 | -5% |
| C2 | -10% |
| C3 | -15% |
| C4 | -20% |
| C5 | -25% |

补充说明：

- 为保持裁判层主结构简洁，`MaxDD` 在基础表中仅按风险等级设定
- 对于退休等对回撤更敏感的客群，可由运行时风控规则在此基础上进一步收紧，作为裁判层之上的 overlay 风控机制
- 因此，`σ` 与 `MaxDD` 分别刻画**常态风险预算**与**极端风险红线**，避免在主表中过度耦合

# 八、risk_anchor.csv 标准结构

建议统一使用：

```csv
profile_id,risk_level,life_stage,sigma_min,sigma_mid,sigma_max,max_drawdown_tolerance
```

字段说明：

- `profile_id`：客户画像主键，对应 `(risk_level, life_stage)` 的组合键，不代表真实客户 ID
- `risk_level`：客户风险等级（C1–C5）
- `life_stage`：客户生命周期阶段
- `sigma_min` / `sigma_mid` / `sigma_max`：客户风险容器
- `max_drawdown_tolerance`：最大回撤容忍度

# 九、体系作用（非常关键）

该体系在评估中承担三件事：

## 1️⃣ 定义“风险容器”

- 每个客户不是一个组合
- 而是一个**风险区间**

---

## 2️⃣ 支撑风险匹配评估

```text
Δσ = Vol_ann - σ_mid
```

用于判断：

- 是否过激（Vol > σ_cap）
- 是否过保守（Vol < σ_min）
- 是否最优匹配（≈ σ_mid）

由于 `Δσ` 基于组合事后实际波动计算，因此它不依赖任何单一策略引擎的目标函数，可作为多引擎框架下统一的客户适配性度量。

---

## 3️⃣ 提供策略优劣判断维度

不仅比较：

- Return
- Sharpe

还比较：

> ❗ **是否适合客户**

---

# 十、关键设计原则（建议写进 README）

## 原则1：不定义标准组合

> 本系统不预设标准资产配置比例，也不绑定任何单一优化范式，仅定义风险承受区间并提供统一评估尺子。

---

## 原则2：风险等级与生命周期解耦

- 风险等级 → 决定风险强度（m_risk）
- 生命周期 → 决定风险上限（σ_stage_max）

其中，`σ_stage_max` 的生命周期递减逻辑可参考 Vanguard / BlackRock 等海外主流 Target-Date Fund 管理机构的 glide path 方法论，并结合本地市场进行校准。

---

## 原则3：风险匹配优于风险最小

> 波动率最低 ≠ 最优  
> 对客户而言，最接近风险预算目标占用率（σ_mid）的组合，才更可能是适配性最优

---

## 原则4：避免双重约束

- σ 控制风险结构
- MaxDD 控制极端风险
- 两者不重复建模

---

## 原则5：风险锚体系是策略无关的裁判层

- 不服务某一个具体优化器的内生求解
- 可同时约束与评估 MVO、风险平价、目标风险、规则混合、专家覆盖等不同引擎
- 核心目标不是寻找单一“数学最优组合”，而是建立统一的客户适配性尺度

---

# 十一、体系边界与局限

- 本体系以年化波动率为核心风险度量，对尾部风险、流动性风险、信用风险的刻画有限，需依赖其他风控模块补充
- `σ_stage_max` 基于历史统计经验、行业共识与本地化校准设定，在市场范式发生明显切换时需要重新评估
- `Δσ` 属于事后风险匹配指标，其评估效果依赖于观察窗口长度；窗口过短时稳定性不足，应结合统一回测窗口与滚动评估机制使用
- 本体系适用于多资产配置组合的统一约束与评估，不适用于高杠杆、单一资产或衍生品主导策略的完整风险刻画
- 本体系解决的是“客户风险适配性”问题，不替代收益预测、择时能力或底层资产研究能力

---

# 十二、一句话总结

> 本体系通过“风险等级 × 生命周期”构建客户风险锚，其中生命周期风险上限参考 Vanguard / BlackRock 等海外主流 Target-Date Fund 管理机构的 glide path 方法论并进行本地化校准，作为智能投顾 3.0 多引擎架构下统一的事前约束与事后裁判层，并以组合实际波动与风险中枢的偏差（Δσ）作为核心适配性指标，实现从“收益比较”向“适配性比较”的升级。
